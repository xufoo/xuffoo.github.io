github+hexo搭建博客出现的问题

Posted on 2018-02-03 | In blog

记录搭建维护博客所出现的问题:

hexo在利用mathjax写公式的时候出现显示错误的问题

原因：hexo默认使用hexo-renderer-marked引擎去渲染网页，它会把利用Markdown语法写的文本去转换为相应的html标签。在利用Markdown写MathJax公式的时候，经常会用到下划线表示下标，但是下划线会被hexo的默认引擎hexo-renderer-marked渲染成html中的\标签，表示斜体，这样一来，我们写的MathJax公式就被错误渲染了，也就没办法正确显示出来
解决方法：换一个渲染引擎 

npm uninstall hexo-renderer-marked --save
npm install hexo-renderer-kramed --save

如果还有部分行内公式显示错误的话可以找到blog/node_modules/kramed/lib/rules/inline.js文件打开编辑

第11行escape: /^\([\`{}[]()#$+-.!_>])/,
将其修改为escape: /^\([`[]()#$+-.!>])/,
第20行em: /^\b((?:_|[\s\S])+?)\b|^*((?:**|[\s\S])+?)*(?!*)/,
将其修改为em: /^*((?:**|[\s\S])+?)*(?!*)/,

hexo 在kali(debian)上安装失败显示无法识别命令

原因： hexo没有写入环境变量
解决办法： 用ln -s 命令
ln -s /home/tools/node/node-v8.9.3-linux-x64/node-v8.9.3-linux-x64/lib/node_modules/hexo-cli/bin/hexo /usr/local/bin/hexo
